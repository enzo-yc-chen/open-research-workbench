# Sample Literature Review

## Goal

Use a small fictional literature review to demonstrate how a task chain can
serve a research project without putting task logs inside the project folder.

## Project Ownership

- Ownership type: project-owned task chain.
- Owner project: [Toy Solver Benchmark](../../projects/toy-solver-benchmark/).
- Project path: `markdown/projects/toy-solver-benchmark/`.
- Research question: Which public source materials are enough to define a
  fair toy-solver benchmark?
- Evidence gap: The project needs one source-backed baseline note and one
  metric sketch before implementation tasks begin.
- Metric or threshold: A useful source must define a reproducible problem,
  input shape, or comparison criterion.
- Failure interpretation: If the reading batch only finds vague motivation, the
  project should stay in planning rather than start benchmark execution.
- Result handback target: `markdown/projects/toy-solver-benchmark/status.md`
  and `markdown/projects/toy-solver-benchmark/analysis/metric-sketch.md`.
